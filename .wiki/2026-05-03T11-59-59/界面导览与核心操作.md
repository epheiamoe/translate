# 界面导览与核心操作

打开 Moe Translate，你首先会被它简洁的三段式布局吸引：顶部是工具栏，中部是标签导航和翻译面板，底部是翻译按钮和信息提示。整个界面由 `App.tsx` 这一根组件驱动，所有子组件通过 props 和全局状态（Zustand store）协同工作。下面我们按从上到下的顺序，逐一认识每个区域。

---

## ❶ 顶部 Header：工具栏

屏幕最顶部的横条就是 **Header**，它承载了所有全局操作入口。Header 的左半部分是「提供商切换」和「模型切换」两个下拉选择框，右半部分是一排图标按钮。

### ProviderSwitcher（提供商切换）

这是一个 `<select>` 下拉框，列出所有可用的 LLM API 提供商。它接收三个 props：

- `currentProvider`：当前选中的提供商 ID
- `providers`：提供商列表，包含内置提供商（如 DeepSeek、OpenAI）和用户自定义提供商
- `onSelect`：切换时触发的回调

当你切换提供商时，`onSelect` 会调用 `getModelsForProvider` 获取该提供商支持的模型列表，并自动选中第一个模型作为默认值。

[来源](src/App.tsx#L139-L153)

### ModelSwitcher（模型切换）

紧邻提供商右侧的 `<select>` 下拉框，列出当前提供商可用的模型。它的 props 是：

- `currentModel`：当前选中的模型 ID
- `models`：模型字典 `Record<string, { name, supports_thinking }>`
- `onSelect`：切换模型时触发

支持 **思考链（reasoning）** 的模型（如 DeepSeek Reasoner）会在名称旁显示 🤖 图标。关于提供商与模型的联动细节，可参阅 [ProviderSwitcher 与 ModelSwitcher 联动机制](providerswitcher-与-modelswitcher-联动机制.md)。

[来源](src/App.tsx#L154-L159)

### 右侧图标按钮组

桌面端（宽度 ≥ 768px）会显示五个图标按钮；移动端则折叠为一个「更多菜单」（三点图标），点击展开下拉菜单。按钮从左到右依次是：

| 图标 | 功能 | 点击行为 |
|---|---|---|
| ⏱️ 历史 | 打开历史面板 | 触发 `setShowHistory(!showHistory)`，渲染 `HistoryPanel` 组件 |
| 📊 统计 | 查看 Token 消耗 | 异步加载 `getTokenStats()`，弹出统计模态框 |
| ⚙️ 设置 | 打开设置面板 | 触发 `setShowSettings(!showSettings)`，渲染 `Settings` 组件 |
| 🎨 样式 | 打开样式自定义 | 触发 `setShowStyleCustomization(true)`，渲染 `StyleCustomization` 组件 |
| 📄 自定义指令 | 打开指令编辑 | 触发 `setShowCustomInstructions(true)`，渲染 `CustomInstructions` 组件 |

所有 Header 按钮共享 CSS 类 `header-btn`，统一为 40×40px 的圆形按钮，悬停时有放大和背景变色效果。

[来源](src/App.tsx#L171-L271)

> **提示**：如果你想了解历史面板的详细操作，可阅读 [历史面板与数据管理](历史面板与数据管理.md)；想配置 API 密钥和提供商，请参考 [API 密钥与提供商配置](api-密钥与提供商配置.md)。

## ❷ Tab 导航：翻译 / 解释 / 文档翻译

Header 下方是 **Tab 导航栏**，包含三个标签页：

- **翻译**（`translate`）：默认模式，输入源语言文本，输出目标语言译文
- **解释**（`explain`）：切换为解析模式，LLM 会对文本进行深度分析而非直接翻译
- **文档翻译**（`doc`）：上传文件或通过 URL 抓取内容，进行长文本/文档翻译

Tab 栏由一个居中的 `tab-indicator` 元素实现滑动指示条动画，切换时指示条通过 CSS `transform: translateX()` 平滑移动。三个 Tab 按钮均分宽度。

切换 Tab 时，`handleTabChange` 函数会同时更新 `activeTab` 和 `mode`——切换到「解释」时自动设为 `parsing` 模式，其他 Tab 设为 `translation` 模式。URL 的 hash 也会同步更新（如 `#/translate`），支持浏览器前进/后退导航。

[来源](src/App.tsx#L98-L113)

关于三种模式的详细区别，请参阅 [翻译模式与解释模式详解](翻译模式与解释模式详解.md)。

## ❸ 翻译面板（核心操作区）

这是你最常使用的区域。当 Tab 为「翻译」或「解释」时，页面主体分为左右两个面板（移动端则上下排列），中间有一个交换按钮。

### 源语言面板（左侧/上方）

**面板标题栏**包含两个控件和一个按钮：

1. **LanguageSelector（语言选择器）** — 源语言选择器
   - 接收 `showAutoOption={true}`，因此第一个选项是「自动检测」
   - 支持自定义语言——你可以在设置中添加不受内置列表支持的语言
   - 下拉菜单显示当前选中项的对勾标记

2. **交换按钮**（仅移动端显示在此处）— 点击后交换源语言和目标语言（当源语言为 `auto` 时禁用）

3. **粘贴按钮（📋）**— 点击后调用 `navigator.clipboard.readText()`，将剪贴板内容填入源文本输入框

**TranslationArea（文本输入区）** 下方的主输入区域，封装了 `<textarea>`，支持受控模式（value + onChange）。它的 props 包括：

- `value`：源文本内容（来自 `useAppStore.getState().sourceText`）
- `onChange`：输入变化时更新 store
- `placeholder`：根据 Tab 显示不同的占位文字（翻译/解释各有独立提示）
- `disabled`：流式翻译进行中时禁用输入

**StyleSettings（风格设置）** 位于源面板底部，默认折叠。展开后可以选择「正式」「休闲」「学术」「文学」「自定义」等预设风格，自定义风格时会出现额外的文本框输入具体指令。

[来源](src/App.tsx#L313-L347)

### 交换按钮（桌面端）

在桌面端，源面板和目标面板之间有一个独立的交换按钮（`swap-btn`），点击后交换源语言和目标语言。流式翻译进行时该按钮禁用。

[来源](src/App.tsx#L349-L367)

### 目标语言面板（右侧/下方）

**面板标题栏**包含一个语言选择器和三个操作按钮：

1. **LanguageSelector（语言选择器）** — 目标语言选择器，`showAutoOption={false}`，因此没有「自动检测」选项
2. **复制按钮** — 将目标文本复制到剪贴板，无文本时禁用
3. **全屏按钮** — 在弹窗中全屏显示翻译结果，便于阅读长文本
4. **Markdown 切换按钮** — 在纯文本和 Markdown 渲染模式之间切换

**翻译结果显示区**有两种呈现方式：

- **Markdown 模式**（默认在「解释」Tab 下启用）：使用 `MarkdownRenderer` 组件渲染，支持代码高亮、表格等 Markdown 语法
- **纯文本模式**：使用 `TranslationArea` 以只读 `readOnly` 方式展示

**思考链折叠区（Thinking Chain）** 当使用的模型支持 reasoning（如 DeepSeek Reasoner）且 LLM 返回了 `reasoning_content` 时，会在目标文本下方显示一个思考链区域。流式翻译过程中，思考内容实时更新；翻译完成后自动折叠为一行按钮，点击可展开查看完整推理过程。

[来源](src/App.tsx#L369-L423)

> 更详细的思考链机制请阅读 [思考链（Thinking Chain）展示组件](思考链-thinking-chain-展示组件.md)。

## ❹ 翻译按钮与 Token 消耗统计

目标面板最下方是主操作按钮：

**翻译按钮**（`.translate-btn`）位于目标面板底部。它的状态跟随 `isStreaming` 变化：

- **就绪状态**：显示「翻译」或「解释」文字，点击调用 `translate()` 函数发起请求
- **流式状态**：按钮变为动画态，左侧出现旋转的 `spinner`，右侧出现滑动展开的红色 STOP 区域，点击可调用 `stopTranslation()` 中止翻译过程
- **禁用条件**：源文本为空，或当前提供商未配置 API Key

按钮的启用条件检查 `safeSettings.providerApiKeys[safeSettings.selectedProvider]`，若未配置 API Key，底部还会弹出黄色警告条引导你打开设置页面。

[来源](src/App.tsx#L428-L447)

**Token 消耗徽标** 在翻译完成后（且 `lastTokenUsage` 有值），按钮下方显示形如 `1,234 tokens ~$0.000012` 的消耗统计。它通过 `getModelConfig` 获取当前模型的定价信息（`pricing_input` 和 `pricing_output`），估算本次翻译的成本（假设输入和输出各占一半 Token）。

[来源](src/App.tsx#L457-L471)

**错误提示** 如果翻译过程中发生错误，会在按钮下方以红色背景显示错误信息。

**替代翻译（Alternatives）** 翻译完成后会出现「显示替代翻译」按钮，点击可触发 `fetchAlternatives()` 获取 3 个不同风格的版本，并可点击任意版本替换当前翻译结果。

[来源](src/App.tsx#L473-L534)

## ❺ 全局覆盖层：弹窗与面板

某些操作会以覆盖层形式弹出：

- **HistoryPanel**：历史记录面板，支持搜索、收藏、删除、导出/导入
- **Settings**：设置面板，管理 API 密钥、默认语言、外观等
- **StyleCustomization**：实时 CSS 变量调色板 + 自定义 CSS 注入
- **CustomInstructions**：自定义指令编辑器，定义术语表和翻译约束
- **全屏结果**：大字体展示翻译结果，带返回按钮
- **Token 统计模态框**：展示总消耗和月度消耗

[来源](src/App.tsx#L536-L569)
