# 快速开始

本指南将带你从零开始，在 5 分钟内完成 Moe Translate 的本地运行并完成第一次翻译。你不需要任何 React 或 TypeScript 基础，只需要基本的命令行操作。

---

## 前置条件

在开始之前，请确保你的计算机上已安装：

- **Node.js 18+**（[下载地址](https://nodejs.org/)）
- **npm 9+**（随 Node.js 一同安装）

打开终端（Windows 上为命令提示符或 PowerShell），验证安装：

```bash
node -v
npm -v
```

你应该看到类似 `v18.x.x` 和 `9.x.x` 的版本号输出。

[来源](../package.json#L14-L23)

---

## 第一步：安装依赖

在工作目录下执行以下命令，安装项目所需的所有依赖包：

```bash
npm install
```

运行成功后，你会看到 `node_modules/` 文件夹被创建，终端输出中没有红色报错信息。

> 💡 如果遇到网络问题，可以尝试使用国内镜像源：`npm config set registry https://registry.npmmirror.com`，然后重新运行 `npm install`。

[来源](../README.md#L68-L72)

---

## 第二步：启动开发服务器

安装完成后，运行：

```bash
npm run dev
```

终端会输出类似以下内容：

```
  VITE v5.x.x  ready in XXX ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

> 🔑 **记住这个地址**：`http://localhost:5173/`，下一步要用。

`npm run dev` 实际上执行的是项目根目录下 `vite.config.ts` 中配置的 Vite 开发服务器。它支持热模块替换（HMR），你修改代码后浏览器会自动刷新。

[来源](../package.json#L9)

---

## 第三步：打开浏览器并进入设置面板

打开浏览器，访问 **http://localhost:5173/**。

你会看到 Moe Translate 的主界面，页面上方还有一个醒目的**黄色警告条**，提示你需要配置 API Key。别担心，我们这就去设置。

点击页面右上角的 **齿轮图标（⚙️）**，打开 **设置面板**（`Settings` 组件）。

![Settings 按钮位置示意]

在设置面板中，你会看到多个配置区域。我们只需要关注最上面名为 **"API Configuration"** 的区域，这里集中了三个关键控件：

| 控件 | 说明 | 类型 |
|------|------|------|
| **Provider 下拉菜单**（`ProviderSwitcher`） | 选择 API 提供商，默认选中 DeepSeek | `<select>` 下拉框 |
| **API Key 输入框** | 输入所选提供商的 API 密钥 | `<input type="password">` 密码输入框 |
| **Model 下拉菜单**（`ModelSwitcher`） | 选择要使用的模型，默认选中 `deepseek-v4-flash` | `<select>` 下拉框 |

这三个控件从上到下排列在 **"API Configuration"** 标题下方，依次是：

1. **Provider**（提供商选择）—— 当前选中 `DeepSeek`
2. **API Key**（API 密钥输入）—— 文本输入框，显示为密码掩码
3. **Model**（模型选择）—— 列表自动随提供商变化

[来源](../src/App.tsx#L196-L208) | [来源](../src/components/Settings/Settings.tsx#L166-L204)

## 第四步：配置 DeepSeek API

### 4.1 确认 API Base URL

Moe Translate 已为 DeepSeek 内置了默认的 API 地址：

```
https://api.deepseek.com/v1
```

你无需手动填写，Provider 下拉菜单选中 `DeepSeek` 时自动生效。如果你使用的是其他兼容 OpenAI 格式的服务商，可以在设置面板下方的 **"Custom Providers"** 区域自行添加。

> 内置提供商列表及定价信息详见：[API 密钥与提供商配置](api-密钥与提供商配置.md)

### 4.2 填入 API Key

1. 在设置面板的 **API Key** 输入框中，粘贴你的 DeepSeek API 密钥
2. 密钥以 `sk-` 开头，可以在 [DeepSeek 官网](https://platform.deepseek.com) 的后台获取
3. 输入框默认显示为密文（密码模式），确保安全

```bash
# API Key 示例格式
sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

每个提供商可以保存独立的 API Key。当你切换 Provider 时，系统会自动加载对应的密钥。[来源](../src/components/Settings/Settings.tsx#L179)

### 4.3 选择模型

在 Model 下拉菜单中，Moe Translate 为 DeepSeek 提供了两个内置模型：

| 模型 | 特点 | 最大上下文 | 支持思考链 |
|------|------|-----------|-----------|
| **DeepSeek Chat** (`deepseek-v4-flash`) | 快速响应，适合日常翻译 | 128K tokens | ❌ |
| **DeepSeek Reasoner** (`deepseek-reasoner`) | 展示推理过程，适合复杂文本 | 128K tokens | ✅ 🤖 |

初次使用选择 **DeepSeek Chat** 即可。如果你需要查看模型的推理过程，可以选择 Reasoner 并开启 Think 模式。模型选择逻辑详见：[ProviderSwitcher 与 ModelSwitcher 联动机制](providerswitcher-与-modelswitcher-联动机制.md)

### 4.4 保存设置

点击设置面板底部的 **"Save"** 按钮，设置会自动保存到浏览器的 IndexedDB 中。黄色警告条随即消失，表示配置成功。

[来源](../src/components/Settings/Settings.tsx#L66-L77) | [来源](../README.md#L86-L91)

## 第五步：输入源文本

回到主界面，你会看到左右或上下两个面板（取决于屏幕宽度）：

- **左侧（或上方）面板**：源文本输入区——`TranslationArea` 组件
- **右侧（或下方）面板**：翻译结果显示区

在左侧的文本框中，输入你想要翻译的文字。例如：

```
Hello, welcome to Moe Translate! This is a powerful offline-first translation tool.
```

你还可以使用面板上的 **语言选择器**（`LanguageSelector`）设定源语言和目标语言：

- 源语言设为 **Auto Detect（自动检测）**
- 目标语言设为 **Chinese (Simplified)**

语言自动检测的原理详见：[语言检测与自定义语言](语言检测与自定义语言.md)

[来源](../src/App.tsx#L259-L281) | [来源](../src/components/TranslationArea/TranslationArea.tsx#L7-L21)

## 第六步：点击翻译按钮

在右侧面板底部，你会看到一个蓝色的 **"Translate"** 按钮。

点击它，开始翻译！

![Translate 按钮位置示意]

翻译过程中你会看到：
1. **按钮变为旋转动画**，显示 "Translating..."
2. **右侧面板** 中逐字出现翻译结果——这是 **流式输出（Streaming）**
3. 翻译完成后，按钮恢复为 "Translate"
4. 页面底部会显示本次翻译消耗的 **Token 数量**（例如 `"50 tokens"`）

流式 API 的底层实现见：[LLM 流式 API 客户端架构](llm-流式-api-客户端架构.md)

🎉 **恭喜！你完成了第一次翻译！**

## 第七步（可选）：构建生产版本

当你完成开发或调试后，可以构建用于生产环境的静态文件：

```bash
npm run build
```

构建过程分为两步（见 `package.json` 中 `build` 脚本的定义）：

1. **TypeScript 编译**（`tsc -b`）：检查类型错误
2. **Vite 打包**（`vite build`）：将 React 代码打包为优化的静态资源

构建成功后，所有产出文件位于项目根目录的 **`dist/`** 文件夹中：

```
dist/
├── index.html              # 入口 HTML
├── assets/
│   ├── index-Czod6a6h.js   # 打包后的 JS（哈希名）
│   └── index-DC9p616G.css  # 打包后的 CSS（哈希名）
├── manifest.webmanifest    # PWA 清单文件
├── registerSW.js           # Service Worker 注册
├── sw.js                   # Service Worker（离线缓存）
├── app-icon-*.png          # PWA 图标
└── _headers / _routes.json # Cloudflare Pages 配置
```

你可以将 `dist/` 目录部署到任意静态文件服务器。推荐阅读 [Cloudflare Pages 部署指南](cloudflare-pages-部署指南.md) 获取详细的部署步骤。

你也可以用 Vite 内置的预览服务器在本地查看生产构建效果：

```bash
npm run preview
```
