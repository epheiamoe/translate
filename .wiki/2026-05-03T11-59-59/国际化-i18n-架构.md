## 国际化（i18n）架构

Moe Translate 使用 **i18next** + **react-i18next** 实现中英文双语界面，通过 JSON 文件管理翻译键值对，并通过 localStorage 持久化用户的语言偏好。整个机制围绕一个简洁的核心文件展开：`src/i18n/index.ts`，它负责初始化、语言检测和语言切换入口。

### 初始化链路：从 import 到 DOM

国际化模块的入口在 `src/main.tsx`，它通过一个简单的 import 引入 i18n 初始化代码：

```typescript
import './i18n';
```

[来源](../src/main.tsx#L4)

这行 import 触发 `src/i18n/index.ts` 的执行。初始化流程分为三个步骤：

1. **确定默认语言** — 按优先级依次检查：`localStorage.getItem('language')` → `navigator.language`（仅提取语言前缀，如 'zh'）→ 回退至 'en'。
2. **设置 HTML lang 属性** — `document.documentElement.lang = defaultLang`，确保浏览器和屏幕阅读器能识别当前界面语言。
3. **启动 i18next** — 通过 `initReactI18next` 插件将 i18n 实例绑定到 React 组件树，注册 `en` 和 `zh` 两套翻译资源，设定 `fallbackLng: 'en'` 作为兜底语言。`interpolation.escapeValue: false` 表示 React 已负责 XSS 防护，i18n 不再重复转义。

[来源](../src/i18n/index.ts#L1-L23)

语言检测的核心逻辑浓缩在以下两行：

```typescript
const savedLanguage = localStorage.getItem('language');
const browserLang = navigator.language.startsWith('zh') ? 'zh' : 'en';
```

[来源](../src/i18n/index.ts#L6-L7)

这意味着：用户一旦手动切换语言，该选择会被持久化，后续访问不再依赖浏览器语言设置。

### 暴露的 API：changeLanguage 与 getCurrentLanguage

`src/i18n/index.ts` 导出两个函数供组件使用：

**`changeLanguage(lang: string)`** — 执行三个动作：将语言代码写入 `localStorage`（键名为 `'language'`），调用 `i18n.changeLanguage(lang)` 触发翻译重新渲染，并同步更新 `document.documentElement.lang`。这种"三合一"设计确保 UI、持久化和无障碍三者一致。

**`getCurrentLanguage(): string`** — 返回 `i18n.language`，即当前激活的语言代码。

[来源](../src/i18n/index.ts#L27-L34)

### 翻译资源文件：嵌套键与占位符插值

两个 JSON 文件位于 `src/i18n/locales/` 目录下，采用一致的嵌套结构：

```
{
  "app": { "title": "Moe Translate" },
  "tabs": {
    "translate": "Translate",
    "explain": "Explain",
    "docTranslate": "Doc Translate"
  },
  "translation": {
    "placeholder": "Enter text to translate..."
  },
  "docTranslation": {
    "incomplete": "Translation incomplete, continuing... ({{current}}/{{max}})",
    "error": "Error: {{message}}"
  },
  ...
}
```

[来源](../src/i18n/locales/en.json) | [来源](../src/i18n/locales/zh.json)

**键的命名风格**：采用 `<模块>.<具体含义>` 的命名空间模式（如 `translation.placeholder`、`settings.apiKey`），便于按功能域查找和避免键名冲突。`zh.json` 包含约 40 个翻译键，覆盖所有 UI 文本。

**占位符插值**：i18next 使用 `{{变量名}}` 语法。以下是从实际代码中提取的三种插值模式：

| 模式 | 源码示例 | 渲染结果（假设变量值） |
|------|---------|----------------------|
| 单变量 | `t('docTranslation.download', { type: 'MD' })` | "Download MD" |
| 多变量 | `t('docTranslation.incomplete', { current: 2, max: 5 })` | "Translation incomplete, continuing... (2/5)" |
| 动态消息 | `t('docTranslation.error', { message: err.message })` | "Error: Connection refused" |

[来源](../src/components/DocumentTranslation/DocumentTranslation.tsx#L237-L251)

### 组件中的 t() 调用模式

在整个组件树中，`t()` 函数的使用遵循以下几种模式：

**1. 静态键名（最常见）**

```tsx
const { t } = useTranslation();
// ...
<h3 className="section-title">{t('settings.apiConfig')}</h3>
<label className="form-label">{t('settings.provider')}</label>
```

[来源](../src/components/Settings/Settings.tsx#L190-L204)

**2. 动态键名（模板字符串）**

LanguageSelector 组件根据语言代码动态构建翻译键名，实现语言名称的自翻译：

```tsx
const translated = t(`languages.${code}`);
return translated !== `languages.${code}` ? translated : languages[code] || code;
```

这种模式利用了 i18next 的一个特性：当键名不存在时，`t()` 返回键名字符串本身。上述代码通过比较返回值与键名来判断翻译是否存在，若不存在则回退到 `languages` 对象中的原始名称。

[来源](../src/components/LanguageSelector/LanguageSelector.tsx#L27-L28)

**3. 带默认值的键名**

Settings.tsx 在一些场景下为每个 `t()` 调用附加了 `||` 回退字符串。这在某些键可能尚未被翻译时提供了额外的安全性：

```tsx
{t('settings.thinkingMode') || 'Thinking Mode'}
{t('settings.promptTranslation') || 'Short Text Translation'}
```

[来源](../src/components/Settings/Settings.tsx#L483-L518)

**4. 条件选择键名**

```tsx
{activeTab === 'explain'
  ? t('explain.placeholder')
  : t('translation.placeholder')}
```

[来源](../src/App.tsx#L266)

### 语言切换的完整链路

用户通过设置界面触发语言切换。以下是完整的调用链：

```
Settings 组件中的语言选择器
  → changeLanguage('zh')  // 从 src/i18n/index.ts 导入
    → localStorage.setItem('language', 'zh')
    → i18n.changeLanguage('zh')
      → react-i18next 重新渲染所有使用 useTranslation() 的组件
    → document.documentElement.lang = 'zh'
```

[来源](../src/i18n/index.ts#L27-L31)

切换后无需手动刷新页面——react-i18next 通过 React Context 传播语言变更，所有使用 `useTranslation()` 的组件自动重新渲染。

### 添加新语言的步骤

要在 Moe Translate 中新增一种语言（例如日语），需要完成以下三个步骤：

**步骤 1：创建新的 JSON 翻译文件**

在 `src/i18n/locales/` 下创建 `ja.json`，复制 `en.json` 的结构，将值翻译为目标语言：

```json
{
  "app": { "title": "萌え翻訳" },
  "tabs": {
    "translate": "翻訳",
    "explain": "説明",
    "docTranslate": "文書翻訳"
  }
  // ... 其他键
}
```

**步骤 2：在 i18n 初始化中注册资源**

修改 `src/i18n/index.ts`，导入新的 JSON 文件并注册到 `resources` 配置中：

```typescript
import ja from './locales/ja.json';

i18n.use(initReactI18next).init({
  resources: {
    en: { translation: en },
    zh: { translation: zh },
    ja: { translation: ja }    // ← 新增
  },
  // ...
});
```

同时需要更新浏览器语言检测逻辑，使其也能识别 `'ja'` 前缀：

```typescript
const browserLang = navigator.language.startsWith('zh') ? 'zh'
  : navigator.language.startsWith('ja') ? 'ja'
  : 'en';
```

**步骤 3：使用 `t()` 替换所有硬编码字符串**

这是确保新旧代码一致的关键。应用中任何硬编码的中文或英文字符串都应替换为 `t()` 调用。查找模式：搜索所有中文字符串（如 `"翻译"`、`"设置"`）或英文 UI 字符串，替换为对应的翻译键。

> 注意：`t('common.more')` 在当前版本中被引用但未在语言文件中定义，这是添加新语言时可以顺便补齐的遗漏项。
> 
> [来源](../src/App.tsx#L529)

### 语言文件的键覆盖范围

`en.json` 和 `zh.json` 目前涵盖以下功能域：

| 命名空间 | 涵盖内容 |
|----------|---------|
| `app` | 应用标题 |
| `languages` | 30 种语言名称（含自动检测） |
| `tabs` | 翻译/解释/文档翻译三种模式标签 |
| `translation` | 翻译输入框、按钮、状态文本 |
| `explain` | 解释模式输入框、结果、按钮 |
| `docTranslation` | 文档翻译全流程文本 |
| `history` | 历史面板标题、搜索、操作按钮 |
| `settings` | API 配置、模型、语言、提示词等设置项 |
| `styleSettings` | 翻译风格选项 |
| `buttons` | 粘贴、复制、交换、全屏等通用按钮 |
| `warnings` | 警告提示（如 API Key 缺失） |
| `thinkingChain` | 思考链展示相关文本 |
| `styleCustomization` | 样式自定义面板文本 |
| `customInstructions` | 自定义指令与术语表文本 |
| `tokenStats` | Token 统计文本 |
| `modelSwitcher` / `providerSwitcher` | 选择器占位文本 |

[来源](../src/i18n/locales/en.json)

### 设计要点总结

| 关注点 | 实现方式 |
|--------|---------|
| **框架** | i18next + react-i18next，经 `initReactI18next` 桥接 |
| **语言检测优先级** | localStorage → navigator.language → 'en' |
| **持久化** | `changeLanguage()` 自动写入 localStorage |
| **无障碍** | 每次语言切换同步更新 `<html lang="...">` |
| **回退机制** | i18next `fallbackLng: 'en'` + 组件内 `||` 默认值双重保障 |
| **插值语法** | `{{变量名}}`，支持单变量和多变量 |
| **资源格式** | 嵌套 JSON，以模块名作为顶层命名空间 |
